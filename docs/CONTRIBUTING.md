# Contributing Guidelines

Thank you for your interest in contributing to Smart-PQC-Gateway! 🚀

---

## How to Contribute

You can contribute in several ways:

* Reporting bugs
* Suggesting features or improvements
* Improving documentation
* Submitting code enhancements
* Adding new research or analysis modules

---

## Getting Started

1. Fork the repository
2. Clone your fork

    ```bash
    git clone https://github.com/PrateekRaj8125/Smart-PQC-Gateway
    ```

3. Navigate to the project directory

    ```bash
    cd Smart-PQC-Gateway/code
    ```

4. Install dependencies

    ```bash
    pip install -r requirements.txt
    ```

---

## Development Guidelines

* Follow clean and readable coding practices
* Use meaningful variable and function names
* Keep functions modular and well-documented
* Avoid breaking existing functionality
* Test your changes before submitting

---

## Commit Guidelines

* Use clear and descriptive commit messages
* Example:

    ```txt
    feat: add PQC latency optimization
    fix: correct packet loss calculation
    docs: update README setup steps
    ```

---

## Pull Request Process

1. Create a new branch

    ```bash
    git checkout -b feature/your-feature-name
    ```

2. Make your changes
3. Commit and push
4. Open a Pull Request

### PR Requirements:

* Clearly describe what was changed and why
* Link related issues (if any)
* Ensure code runs without errors

---

## Code Style

* Python: Follow PEP 8
* JavaScript: Use consistent formatting
* Keep UI changes aligned with existing design

---

## Areas for Contribution

* AI model improvements
* Real PQC integration (future work)
* Performance optimization
* Visualization enhancements
* Security improvements

## Questions?

Feel free to open an issue for clarification or discussion.
