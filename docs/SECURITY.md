## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

### 📧 Contact:

[prateek.raj.indt@gmail.com](mailto:prateek.raj.indt@gmail.com)

---

## Reporting Guidelines

When reporting a vulnerability, include:

* Description of the issue
* Steps to reproduce
* Potential impact
* Suggested fix (if available)

---

## Response Process

* Acknowledgement within 48 hours
* Investigation and validation
* Fix development and testing
* Responsible disclosure after resolution

---

## Scope of Security

This project is a **research/demo system**, so:

* Cryptographic implementations are partially simulated
* PQC is not production-grade
* Network conditions are intentionally simulated

Despite this, we take security concerns seriously.

---

## Best Practices for Users

* Do not use this system for real sensitive data
* Run locally or in controlled environments
* Keep dependencies updated
* Avoid exposing the server publicly without safeguards

---

## Responsible Disclosure

Please do not publicly disclose vulnerabilities until they are resolved.
